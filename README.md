## [ 주요 코드 구조 ]

### ActionQueue
- 카드 게임의 동적인 게임플레이 플로우를 지원하기 위해 모든 진행 로직을 IBattleAction으로 추상화
- 액션을 큐 기반으로 관리하고 await를 통해 순차 실행하도록 설계
- Append / Prepend API를 제공하여 액션 실행 순서를 명시적으로 제어
- 반격, 사망 처리 등 우선 수행되어야 하는 이벤트를 기존 플로우에 자연스럽게 삽입 가능

### Model / View
- Board, BoardSide, CardInstance를 MonoBehaviour를 상속하지 않는 순수 데이터(Model)로 구현하여 View와 분리
- 입력 계층은 View를, View는 Model을 참조하도록 설계하여 의존성을 단방향으로 유지
- 표현 방식 변경 시 데이터 클래스 수정이 불필요한 구조 구현 (데이터 로직과 렌더링 이슈를 분리하여 버그 원인 추적 용이)

### Data
- 카드와 덱을 인스펙터에서 관리 가능하도록 ScriptableObject 기반으로 구현
- OnDeployed / OnTurnStart / OnTurnEnd / OnUsed 훅을 정의하여 카드 효과 구현을 추상화
- OnUsed를 활용한 인접 카드 추가 공격, OnTurnStart를 활용한 턴 시작 시 아군 회복 등 다양한 효과 구현 가능

## [ 구현한 기능 목록 ]

### 턴 진행  
턴 시작 → 카드 사용 → 턴 종료  
플레이어와 적이 번갈아가며 턴을 가지며, 카드 사용 시 자동으로 턴이 종료

### 카드 선택 및 공격
필드에 배치된 카드 중 하나를 터치 및 상대방의 카드로 드래그하여 공격

### 카드별 효과 처리
<카드 배치 · 턴 시작 · 카드 사용 · 턴 종료> 각 단계에 따라 카드별 효과를 추가할 수 있음

### 카드 제거 처리
공격을 통해 HP가 0이 되면 필드에서 카드 제거

### 신규 카드 자동 배치
플레이어/적 턴이 시작되기 전, 필드에 빈 슬롯이 있으면 그 수만큼 카드 자동 배치

### 승리 및 패배 판정
적의 모든 카드를 제거했으면 승리, 플레이어의 모든 카드가 제거되었으면 패배 처리

### 적 AI
적은 체력이 가장 높은 카드를 선택하여 체력이 가장 낮은 플레이어의 카드를 타겟팅하여 공격  
(체력이 가장 높은 카드 또는 체력이 가장 낮은 카드의 후보가 2장 이상일 때는 그중에 랜덤 선택)

### 연출
VFX, SFX, 트윈 애니메이션, 타겟팅 화살표
